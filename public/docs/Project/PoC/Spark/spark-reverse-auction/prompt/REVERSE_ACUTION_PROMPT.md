### 역경매 소개 페이지 (Landing Page) 설계 요청

다음 조건을 만족하는 역경매 플랫폼 랜딩 페이지를 설계해줘.

#### 페이지 목표
- 첫 방문자가 3초 안에 "역경매가 뭔지, 어떤 이득이 있는지" 이해할 수 있어야 함
- 감정 자극: "견적 받기 힘들었던 경험" → "이제는 전문가들이 먼저 경쟁" → "나는 그냥 선택만"
- 즉시 행동 유도: "무료로 견적 요청하기" CTA 버튼 강조

#### 섹션 구성 제안
1. **Hero Section** (첫 화면)
   - 강렬한 헤드라인:
     - 예: "전문가들이 내 조건에 맞춰 가격 경쟁합니다"
     - 예: "견적 10곳 받으려고 전화할 필요 없어요"
   - 서브 헤드라인: 역경매 핵심 가치 (시간 절약, 최저가 보장, 투명한 비교)
   - CTA 버튼: "지금 무료로 견적 요청" (눈에 확 띄는 색상, 애니메이션)
   - 배경: 실제 시공/작업 사진 또는 그래픽 일러스트 (신뢰감 + 현대적)
   - 우측 또는 하단: 간단한 통계 배지
     - "평균 20% 비용 절감"
     - "전문가 500+ 참여 중"
     - "24시간 내 견적 도착"

2. **작동 방식 (How It Works)** - 3단계 인포그래픽
   - Step 1: "조건 입력" (폼 아이콘 + "2분이면 완료")
   - Step 2: "전문가 입찰" (시계 + 하향 화살표 아이콘 + "실시간으로 가격 내려감")
   - Step 3: "비교 후 선택" (체크 아이콘 + "리뷰, 포트폴리오 확인 후 결정")
   - 각 단계마다 짧은 설명 문구 + 아이콘/일러스트

3. **실시간 역경매 예시 (Live Demo or Mock)**
   - 실제 역경매 진행 중인 화면을 미리보기로 보여주거나,
   - 애니메이션으로 "₩500,000 → ₩480,000 → ₩450,000" 숫자가 실시간으로 떨어지는 연출
   - "지금 이 순간에도 OO명이 견적을 받고 있어요" 실시간 카운터

4. **신뢰 요소 (Trust Signals)**
   - 실제 사용자 후기 (별점 + 짧은 리뷰 3-5개)
   - 제휴 파트너/인증 배지 (있다면)
   - 보안/결제 안전 표시

5. **카테고리 소개**
   - 인테리어, 사진/영상, 자동차 수리, 이사, 청소 등 아이콘 그리드
   - 각 카테고리 클릭 시 해당 분야 전문가 수 + 평균 견적 소요 시간 표시

6. **CTA 반복 (Final Push)**
   - 페이지 하단에 다시 한번 "무료 견적 요청" 버튼
   - "회원가입 없이 바로 시작" 강조

7. **FAQ (접기/펼치기)**
   - "역경매가 정확히 뭔가요?"
   - "전문가들은 어떻게 선정되나요?"
   - "견적 요청은 정말 무료인가요?"
   - "개인정보는 안전한가요?"

#### 디자인/UX 요구사항

### 역경매 진행 페이지 (실시간 입찰 화면) 설계 요청

고객이 견적 요청을 올린 후, 전문가들이 실시간으로 가격을 내려가며 경쟁하는 화면을 설계해줘.

#### 페이지 목표
- **실시간성 강조**: WebSocket 통해 새 입찰이 들어올 때마다 즉각 반영, 애니메이션
- **비교 편의성**: 여러 전문가의 입찰을 한눈에 비교 (가격, 리뷰, 경력, 포트폴리오)
- **긴장감 조성**: 카운트다운, "새로운 입찰!" 알림, 가격 하락 애니메이션으로 흥미 유발
- **신뢰 구축**: 전문가 프로필, 리뷰, 인증 배지 명확히 표시
- **행동 유도**: "이 전문가 선택하기" 버튼 명확

#### 화면 레이아웃 (Desktop 기준)

**좌측 영역 (30-40% 너비)**
- **요청 정보 카드** (고정)
  - 카테고리 (예: 인테리어 > 주방 리모델링)
  - 희망 예산 범위
  - 작업 일정
  - 상세 조건 (접기/펼치기)
  - 첨부 사진/도면 (있다면)
- **진행 상태**
  - 남은 시간 카운트다운 (예: "23:45:12 남음")
  - 현재 입찰 수 (예: "8명의 전문가가 제안 중")
  - 최저가 (큰 폰트로 강조, 실시간 업데이트)

**우측 영역 (60-70% 너비)**
- **입찰 리스트** (실시간 업데이트)
  - 카드 형태로 각 전문가 입찰 표시
  - 정렬 옵션: "가격 낮은 순" (기본), "평점 높은 순", "최근 입찰 순"
  - 각 카드 구성:
    ```
    [프로필 사진] [전문가명] [인증 배지]
    ⭐ 4.8 (리뷰 127개) | 경력 8년 | 완료 프로젝트 243개
    
    제안 금액: ₩450,000 (방금 전 ₩480,000에서 하락 ↓ 빨간색 애니메이션)
    예상 기간: 2주
    
    [한 줄 소개/차별점]
    "친환경 자재 전문, 무상 A/S 1년 제공"
    
    [상세보기] [선택하기] 버튼
    ```

#### 실시간 기능 요구사항

1. **WebSocket 연동** (내 SDK 사용) [memory:33]
   - 페이지 진입 시 SDK 초기화 + 해당 역경매 room join
   - 이벤트 구독:
     - `new_bid`: 새 입찰 도착 → 리스트 업데이트 + 알림
     - `bid_updated`: 기존 전문가가 가격 수정 → 해당 카드 강조
     - `auction_ended`: 마감 → "마감되었습니다" 오버레이 + 최종 선택 유도
   - 서버 시각 동기화로 남은 시간 정확히 표시

2. **실시간 UI 애니메이션** (감정 자극!)
   - 새 입찰 도착 시:
     - 카드가 리스트 상단에 fade-in + slide-down
     - "🔥 새로운 제안!" 배지 3초간 표시
     - 효과음 (선택 사항, 토글 가능)
   - 가격 하락 시:
     - 금액 숫자 빨간색으로 깜빡임
     - 하향 화살표 아이콘 (↓) 애니메이션
     - 이전 가격 취소선 표시 (예: ~₩480,000~ → ₩450,000)
   - 최저가 갱신 시:
     - 좌측 "최저가" 섹션에서 confetti 또는 pulse 애니메이션
     - "새로운 최저가!" 토스트 알림

3. **카운트다운 타이머**
   - 좌측 고정 영역에 큰 폰트로 표시
   - 10분 남았을 때 노란색, 5분 남았을 때 빨간색 강조
   - 마감 시 "종료" 애니메이션 + 자동으로 "선택하기" 유도 모달

4. **필터/정렬**
   - 가격 범위 슬라이더 (예: ₩400k - ₩600k)
   - 평점 필터 (4.5★ 이상만 보기)
   - 인증 배지 필터 (공식 인증, 보험 가입 등)

#### 전문가 상세보기 모달/사이드패널
- 입찰 카드에서 "상세보기" 클릭 시 열림
- 구성:
  - 프로필 사진 + 자기소개
  - 별점 + 최근 리뷰 3개
  - 포트폴리오 갤러리 (이전 작업 사진 3-5장)
  - 경력/자격증/인증 배지
  - 제안 내용 상세 (자재, 공정, 일정 등)
  - "이 전문가 선택" CTA 버튼 (크고 명확)

#### 선택 플로우
- "선택하기" 버튼 클릭 시:
  1. 확인 모달: "OOO 전문가를 선택하시겠습니까?"
  2. 선택 완료 시:
     - 해당 전문가에게 알림 전송 (WebSocket)
     - 다른 전문가들에게 "선택 완료" 상태 업데이트
     - 고객은 채팅/상담 페이지로 이동 (또는 결제 프로세스 시작)

#### 엣지 케이스 처리
- 입찰이 0개일 때: "아직 제안이 없어요. 조금만 기다려주세요!" + 로딩 애니메이션
- 입찰 마감 후 접속 시: "이미 종료된 역경매입니다" + 최종 결과 요약
- WebSocket 연결 끊김 시: "실시간 연결 재시도 중..." 배너 + 재연결 로직
- 동시 선택 방지: 서버 단 처리 + 클라이언트는 "이미 다른 고객이 선택했습니다" 에러 처리

#### 감정 자극 포인트
- **경쟁 심리 활용**: "현재 8명이 제안 경쟁 중!" → 전문가들이 치열하게 경쟁하는 느낌
- **실시간 긴장감**: 카운트다운 + 가격 하락 애니메이션 → "지금 선택하지 않으면 기회 놓칠 수도"
- **투명성**: 모든 입찰 공개 + 리뷰 공개 → 신뢰감
- **선택의 주도권**: "내가 고르는 입장" → 고객 만족도 상승
- **FOMO**: "다른 고객이 먼저 선택할 수 있습니다" (선택 시 해당 전문가는 다른 역경매에서 우선순위 하락 등)

#### 디자인/UX 요구사항
- **색상**:
  - 새 입찰/가격 하락: 빨간색/주황색 (긴급함, 흥미)
  - 최저가: 파란색/초록색 (신뢰, 안정)
  - 선택 버튼: 진한 파란색 또는 초록색 (행동 유도)
- **카드 디자인**:

  - 선택된 카드는 테두리 강조
- **애니메이션**:
  - framer-motion으로 리스트 아이템 stagger 애니메이션
  - 숫자 변경 시 countUp/countDown 효과
  - 토스트 알림: react-hot-toast 또는 sonner
- **반응형**:
  - 모바일: 좌측 영역은 상단에 접을 수 있는 드로어
  - 입찰 리스트는 세로 스크롤 (카드는 전체 너비)
